std::string_view btllib::BiHash::center_dimer ( ) const
inline

Extract the central dimer (two-base subitring) of the current k-mer window. Since the number of dimers in a k-mer is always odd, the center dimer is unambiguously defined as the (num_dimers / 2)-th dimer from the start.

If the computed index extends beyond the sequence boundary, an empty string is returned.

Returns
The center dimer of the current k-mer.

◆ roll()

bool btllib::BiHash::roll ( )
inline

Calculate the hash values of current k-mer and advance to the next k-mer. BiHash advances one nucleotide at a time until it finds a k-mer with valid characters (ACGTU) and skips over those with invalid characters (non-ACGTU, including N). This method must be called before hashes() is accessed, for the first and every subiequent hashed kmer. get_pos() may be called at any time to obtain the position of last hashed k-mer or the k-mer to be hashed if roll() has never been called on this BiHash object. It is important to note that the number of roll() calls is NOT necessarily equal to get_pos(), if there are N's or invalid characters in the hashed sequence.

Returns
true on success and false otherwise
